This guide walks you through adding VS2 as a Gradle dependency and making your first API calls. VS2 exposes a stable Kotlin/Java API through the `VsApi` interface, available via the `vsApi` top-level property.

<Note>
  VS2 is currently on **2.5.0-beta.3** for Minecraft 1.20.1. On Forge development environments, VS2 also requires KotlinForForge to be present.
</Note>

## Adding the Maven repository

Add the Valkyrien Skies Maven repository to your `build.gradle` or `build.gradle.kts`:

<CodeGroup>

```groovy build.gradle
repositories {
    // ...
    maven {
        name = "Valkyrien Skies"
        url = "https://maven.valkyrienskies.org"
    }
    // ...
}
```

```kotlin build.gradle.kts
repositories {
    // ...
    maven {
        name = "Valkyrien Skies"
        url = uri("https://maven.valkyrienskies.org")
    }
    // ...
}
```

</CodeGroup>

## Adding the dependencies

Add VS2 and the VS Core libraries to your `dependencies` block. Replace `-//loader//` with your mod loader (`-fabric`, `-quilt`, or `-forge`), or use `-common` if you are in a multiloader project on the Common platform.

<CodeGroup>

```groovy build.gradle
dependencies {
    // ...
    modApi("org.valkyrienskies:valkyrienskies-120-//loader//:${vs2_version}") {
        exclude group: 'com.simibubi', module: ''
        exclude group: 'dev.engine-room', module: ''
        exclude group: 'com.jozufozu', module: ''
    }
    implementation("org.valkyrienskies.core:api:${vscore_version}") {
        exclude group: 'org.joml', module: ''
    }
    implementation("org.valkyrienskies.core:internal:${vscore_version}") {
        exclude group: 'org.joml', module: ''
    }
    implementation("org.valkyrienskies.core:util:${vscore_version}") {
        exclude group: 'org.joml', module: ''
    }
    // ...
}
```

```kotlin build.gradle.kts
dependencies {
    // ...
    modApi("org.valkyrienskies:valkyrienskies-120-//loader//:${vs2_version}") {
        exclude(group = "com.simibubi")
        exclude(group = "dev.engine-room")
        exclude(group = "com.jozufozu")
    }
    implementation("org.valkyrienskies.core:api:${vscore_version}") {
        exclude(group = "org.joml")
    }
    implementation("org.valkyrienskies.core:internal:${vscore_version}") {
        exclude(group = "org.joml")
    }
    implementation("org.valkyrienskies.core:util:${vscore_version}") {
        exclude(group = "org.joml")
    }
    // ...
}
```

</CodeGroup>

Define the versions in your `gradle.properties`. You can find the current `vscore_version` in the VS2 repository's own `gradle.properties`. The VS2 version follows the pattern `version+beta+first10charsOfCommitHash`.

```properties gradle.properties
vs2_version=2.5.0-beta.3+<commit-hash>
vscore_version=<vscore-version>
```

<Warning>
  The excludes on `com.simibubi`, `dev.engine-room`, and `com.jozufozu` prevent VS2's bundled Create-compatibility dependencies from conflicting with Create if you also depend on it. Do not remove them unless you know what you are doing.
</Warning>

## Accessing the API

The VS2 API is exposed through the `VsApi` interface. The simplest way to access it is to import the top-level `vsApi` property from `org.valkyrienskies.mod.api`:

```kotlin
import org.valkyrienskies.mod.api.vsApi

// vsApi is a lazy singleton — safe to import and use anywhere after mod init
val api: VsApi = vsApi
```

`vsApi` is initialized lazily on first access. If VS2 is not installed when it is first accessed, an `IllegalStateException` is thrown with a helpful message.

## Using extension functions

VS2 provides Kotlin extension functions on `Level`, `MinecraftServer`, `Minecraft`, and other Minecraft types. These are defined in `org.valkyrienskies.mod.api.ValkyrienSkies` and are the recommended way to interact with the API from Kotlin.

### Getting the ship world

```kotlin
import org.valkyrienskies.mod.api.shipWorld

// Server side
val shipWorld = server.shipWorld          // ServerShipWorld?

// Client side
val shipWorld = Minecraft.getInstance().shipWorld  // ClientShipWorld?

// From a Level
val shipWorld = level.shipWorld           // ShipWorld?
```

### Querying ships by block or chunk

```kotlin
import org.valkyrienskies.mod.api.getShipManagingBlock
import org.valkyrienskies.mod.api.getShipManagingChunk
import net.minecraft.core.BlockPos

// Get the ship whose shipyard contains a given block position
val ship = level.getShipManagingBlock(BlockPos(100, 64, 200))

// Get the ship whose shipyard contains a given chunk
val ship = level.getShipManagingChunk(chunkX = 6, chunkZ = 12)
```

### Getting all ships in a level

```kotlin
import org.valkyrienskies.mod.api.shipWorld

val allShips = level.shipWorld?.allShips
allShips?.forEach { ship ->
    println("Ship ${ship.id} is at ${ship.transform.positionInWorld}")
}
```

### Checking whether a block is in a shipyard

```kotlin
import org.valkyrienskies.mod.api.isBlockInShipyard

val inShipyard = level.isBlockInShipyard(BlockPos(100, 64, 200))
```

### Finding ships near a world position

```kotlin
import org.valkyrienskies.mod.api.getShipsIntersecting

val nearbyShips = level.getShipsIntersecting(x = 0.0, y = 64.0, z = 0.0)
```

## Java interop

All API entry points are accessible from Java. The `vsApi` property is exposed as a static method via its `@JvmName("api")` annotation:

```java
import org.valkyrienskies.mod.api.ValkyrienSkies;
import org.valkyrienskies.mod.api.VsApi;

VsApi api = ValkyrienSkies.api();
```

Static helpers on `Level` and `MinecraftServer` are available as static methods on the `ValkyrienSkies` class when called from Java.

<Tip>
  Avoid using any class or item with "Debug" in its name. Debug items and classes are internal development tools and are not part of the stable API.
</Tip>
